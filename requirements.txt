================================================================================
  1. TECH STACK & DEPENDENCIES
================================================================================

FRAMEWORK:
  - Next.js 14 (App Router, TypeScript, strict mode)

STYLING:
  - Tailwind CSS (mobile-first utility classes)
  - CSS Variables for theming

ANIMATION:
  - Framer Motion (all transitions, scroll reveals, stagger effects)
  - canvas-confetti (celebration bursts on publish + page load)

STATE MANAGEMENT:
  - zustand (global form state, template selection, preview sync)
  - react-hook-form + zod (all form validation)

UTILITIES:
  - nanoid (unique URL slug generation)
  - date-fns (countdown timer, date formatting)
  - react-hot-toast (toast notifications)

FONTS (next/font/google):
  - "Cormorant Garamond" — romantic serif for all section headings
  - "Great Vibes"         — cursive script for couple names & monograms
  - "DM Sans"             — clean body text, labels, buttons
  - "Noto Sans Tamil"     — Tamil language support on invitation pages
  - "Fira Code"           — URL display in builder

NPM INSTALL COMMAND:
  npm install framer-motion zustand react-hook-form zod @hookform/resolvers
              nanoid date-fns canvas-confetti react-hot-toast

  - All 6 invitation templates respect the chosen theme's color variables

================================================================================
  5. TYPOGRAPHY SYSTEM
================================================================================

FONT ROLES:
  Great Vibes        : Couple names, monograms, romantic headings (cursive)
  Cormorant Garamond : Section headings, event names, quotes (serif)
  DM Sans            : Body text, labels, nav, buttons, captions (sans-serif)
  Noto Sans Tamil    : Tamil/regional language text (when language = 'ta' etc.)

SIZE SCALE (mobile → tablet → desktop):

  Couple Names (Great Vibes):
    Mobile  : 72px–80px
    Tablet  : 96px–110px
    Desktop : 120px–140px
    MINIMUM : Never below 48px even on smallest mobile (looks bad)

  Section Headings (Great Vibes):
    Mobile  : 40px–52px
    Tablet  : 52px–64px
    Desktop : 56px–72px

  Subheadings (Cormorant Garamond):
    Mobile  : 24px–32px
    Tablet  : 28px–36px
    Desktop : 32px–52px

  Body Text (DM Sans):
    Mobile  : 14px–16px
    Tablet  : 15px–17px
    Desktop : 15px–18px

  Countdown Numbers (Cormorant Garamond):
    Mobile  : 48px–56px
    Tablet  : 56px–64px
    Desktop : 64px–80px

  Monogram Closing (Great Vibes):
    Mobile  : 80px–96px
    Tablet  : 96px–110px
    Desktop : 120px–140px

FLUID TYPOGRAPHY:
  Use clamp() for all major text sizes:
    Couple name: clamp(72px, 12vw, 140px)
    Section heading: clamp(40px, 6vw, 72px)
    Body: clamp(14px, 1.5vw, 18px)

  5xl    : 1920px (full HD monitors)

  screens: {
    'xs': '320px',
    'sm': '375px',
    'md': '768px',
    'lg': '1024px',
    'xl': '1280px',
    '2xl': '1440px',
  }

LAYOUT STRATEGY:
  Mobile First — base styles = mobile, media queries scale UP
  Touch-first interactions — hover effects are enhancement only
  No content hidden on any breakpoint — everything accessible on all devices

================================================================================
  12. API ROUTES
================================================================================

POST /api/weddings/create
  Input: full wedding form data (validated with zod)
  Process: generate unique slug (nanoid 8 chars), save to MongoDB
  Output: { success, slug, previewUrl }

GET /api/weddings/[slug]
  Input: slug URL param
  Process: fetch from MongoDB by slug, increment views counter
  Output: full wedding object OR 404

POST /api/rsvp/[slug]
  Input: { guestName, guestCount, mealPreference, message }
  Process: validate, save RSVP to MongoDB
  Output: { success, message }

GET /api/rsvp/[slug]
  Input: slug + auth token header
  Process: verify couple auth, fetch all RSVPs for slug
  Output: array of RSVP objects

POST /api/check-slug
  Input: { slug }
  Process: check MongoDB for existing slug
  Output: { available: boolean, suggestion: string }
